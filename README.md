- [使用Figma构建组件库（2024更新版）](https://www.smashingmagazine.com/2024/01/building-component-library-figma/)

- [Figma组件架构最佳实践](https://www.figma.com/best-practices/component-architecture/)

- [Figma变量（Variables）完全指南](https://www.figma.com/blog/figma-variables/)  
  核心收获：官方详解变量的创建、关联和批量更新，解决「改一个颜色/字体要改全图」的痛点，附设计系统中的变量用法案例。
- [Stack镜像：为代码设计，还是为设计编写代码？](https://www.designsystems.com/stack-mirroring-designing-for-code-and-coding-for-design/)

## 四、开源插件（补充热门工具，通俗化说明用途）
精选「高频使用+维护活跃」的插件，附「一句话用途」方便理解。

- [Themer](https://github.com/thomas-lowry/themer)  
  用途：一键切换设计稿的样式主题（如从「白天模式」切「黑夜模式」），支持批量更新颜色、字体。
- [Figlight](https://github.com/jeetiss/figlight)  
  用途：在Figma中插入代码片段并自动高亮语法（支持JS、CSS、Python等），适合设计开发文档类界面。
- [FigmaThird](https://github.com/ahkohd/FigmaThird)  
  用途：直接在Figma里导入3D模型（支持GLB/GLTF格式），可调整角度和光影，适合做3D界面元素设计。
- [svg-to-jsx](https://github.com/SaraVieira/svg-to-jsx)  
  用途：把Figma中的SVG图标/图形，直接转换为React组件代码，开发不用手动写SVG属性。
- [figma-latex-complete-plugin](https://github.com/maxkrieger/figma-latex-complete-plugin)  
  用途：在Figma中插入LaTeX公式（如数学公式、物理符号），适合设计教育、科研类产品界面。
- [system.colors](https://github.com/thelittlewonder/system.colors)  
  用途：一键导入热门设计系统的调色板（如Material Design、Tailwind、Ant Design），省去手动吸色。
- [FigmaToCode](https://github.com/bernaferrari/FigmaToCode)  
  用途：将Figma设计稿生成可直接用的代码（支持Tailwind CSS、Flutter、SwiftUI），大幅缩短开发还原时间。
- [tailwindcss-figma-plugin](https://github.com/impulse/tailwindcss-figma-plugin)  
  用途：在Figma中直接调用Tailwind的样式类（如padding、color），确保设计与Tailwind开发规范一致。
- [Iconify](https://github.com/iconify/figma-iconify)  
  新增热门插件：集成100+图标库（共100万+图标），可直接搜索插入，支持自定义颜色和大小。
- [Auto Layout Pro](https://github.com/yuanqing/figma-auto-layout-pro)  
  新增热门插件：强化Figma原生自动布局功能，支持「等间距分布」「对齐吸附」等进阶操作，排版更高效。


## 五、插件开发（补充开发入口，明确工具用途）
面向开发者的工具包与模板，降低Figma插件开发门槛。

- [Figma插件开发官方文档](https://www.figma.com/developers/docs)  
  核心价值：入门必备，涵盖插件API、开发流程、调试方法，附TypeScript支持教程。
- [react-figma](https://github.com/react-figma/react-figma)  
  用途：用React语法开发Figma插件，熟悉React的开发者可快速上手，无需重新学Figma原生API。
- [create-figma-plugin](https://github.com/yuanqing/create-figma-plugin)  
  用途：Figma插件开发的「脚手架工具」，自带构建、打包、发布流程，支持React和TypeScript。
- [figplug](https://github.com/rsms/figplug)  
  用途：轻量的插件构建工具，支持热重载（改代码实时看效果），适合开发简单插件。
- [figma-plugin-react-vite](https://github.com/CoconutGoodie/figma-plugin-react-vite)  
  用途：基于React+Vite的插件模板，比传统模板构建速度更快，适合开发复杂交互的插件。
- [figma-plugin-helpers](https://github.com/figma-plugin-helper-functions/figma-plugin-helpers)  
  用途：封装了大量常用工具函数（如获取选中元素、批量修改样式），避免重复写基础代码。

## 七、Rest API 集成（优化描述，补充实用工具）
连接Figma与开发流程的工具，实现「设计资源自动化同步」。

- [storybook-addon-figma](https://github.com/hharnisc/storybook-addon-figma)

- [Figma.js](https://github.com/jongold/figma-js)  
  用途：Figma API的JavaScript封装，开发者可通过代码批量获取设计稿中的颜色、组件、图片。
- [figma-export](https://github.com/RedMadRobot/figma-export)  
  用途：命令行工具，自动将Figma中的颜色、字体、图标导出为iOS/Android项目支持的格式（如Assets.xcassets）。
- [Figma to Flutter](https://github.com/aloisdeniel/figma-to-flutter)  
  用途：将Figma组件转换为Flutter代码（含Widget结构和样式），减少移动端开发的重复工作。
- [FigmaPy](https://github.com/Amatobahn/FigmaPy)  
  用途：Figma API的Python封装，适合用Python做自动化脚本（如批量检查设计规范、导出设计报告）。
- [figma-slack-updates](https://github.com/jordansinger/figma-slack-updates)  
  用途：Figma文件更新后自动推送到Slack频道，团队成员实时知晓设计变更。
- [Figma资产下载器](https://robertohuertasm.github.io/figma-asset-downloader)  
  用途：在线工具，可批量下载Figma中的图标、图片并自动压缩，优化资源加载速度。


## 八、客户端工具（保留活跃工具，补充官方选项）
提供更贴合不同系统的Figma使用体验。

- [Figma官方桌面端](https://www.figma.com/downloads/)  
  优势：官方推出，支持Mac/Windows，相比浏览器版更稳定，可离线编辑（联网后自动同步）。

- [figma-linux](https://github.com/ChugunovRoman/figma-linux)  
  优势：非官方Linux客户端（基于Electron），解决Linux用户无法使用官方桌面端的问题。


## 九、转换工具（更新实用选项，明确转换场景）
实现Figma与其他工具的格式互通。

- [Aeux](https://aeux.io/)  
  用途：将Figma设计稿（含图层结构、样式）导入Adobe After Effects，快速制作动效视频。
- [Magicul](https://magicul.io)  
  用途：将Figma文件转换为Sketch/Adobe XD格式，适合团队内同时使用多种设计工具的场景。
- [Figma to Lottie](https://www.figma.com/community/plugin/841133452981258160/LottieFiles)  
  用途：将Figma中的动效（如按钮点击、页面过渡）转换为Lottie格式，可直接嵌入网页/APP。
- [Figma to PDF](https://www.figma.com/community/plugin/739027014833456083/Figma-to-PDF)  
  用途：批量将Figma页面导出为PDF，支持自定义尺寸和压缩质量，适合交付设计方案文档。
